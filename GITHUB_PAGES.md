# GitHub Pages 공개 배포

이 프로젝트에는 `client` 폴더를 GitHub Pages로 게시하는 자동 배포 작업이 포함되어 있다.

1. GitHub에서 공개 저장소를 만들고 이 프로젝트를 `main` 브랜치에 올린다.
2. 저장소의 `Settings → Pages`에서 배포 원본을 `GitHub Actions`로 선택한다.
3. `Actions → Deploy BOXHEAD to GitHub Pages` 작업이 완료될 때까지 기다린다.
4. 배포 작업의 `deploy` 단계 또는 `Settings → Pages`에 표시된 공개 주소를 연다.

이후 `main` 브랜치에 변경 사항을 올릴 때마다 웹 게임이 자동으로 갱신된다. Windows 오프라인판은 GitHub Release에 완성 ZIP을 첨부하는 방식이 적합하다.

저장소 주소를 직접 열면 소스 목록이 보이는 것이 정상이며 게임 주소와는 다르다. Pages 주소는 보통 `https://계정.github.io/저장소/` 형식이다. 이 프로젝트의 루트 `index.html`은 브랜치 루트 방식으로 Pages를 게시한 경우에도 `client/` 게임으로 즉시 이동한다.

모바일에서 Pages 주소를 열면 터치 기기를 자동 감지한다. 게임 시작 버튼을 누르면 지원 브라우저에서 전체 화면·가로 방향 잠금을 요청하며, 지원하지 않는 브라우저의 세로 화면에는 회전 안내가 표시된다.

v47부터 핵심 웹 파일은 서비스 워커에 캐시된다. 첫 접속은 인터넷이 필요하지만 이후에는 브라우저가 저장 공간을 지우지 않는 한 오프라인 재실행도 가능하다. 새 버전 배포 뒤에는 한 번 새로고침하면 v47 캐시로 갱신된다.
