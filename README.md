## 바로 실행

압축을 푼 뒤 아이콘이 적용된 `BOXHEAD.exe` 또는 `BOXHEAD_게임실행.bat`를 더블클릭한다. Node.js 설치, `npm install`, 서버 주소 입력은 필요 없다. 오래된 사무용 PC에서는 시작 화면의 그래픽 설정이나 `ESC → 그래픽 품질`에서 `초저사양 / 사무용 PC`를 선택한다. 초저사양 전용 실행기는 없으며, 이 모드는 게임 설정에서만 켜고 끈다.

## 그래픽 모드

- 자동 감지: CPU 스레드, 추정 메모리, 화면 해상도로 품질을 선택한다.
- 초저사양: 30 FPS, 30~44% 동적 내부 해상도, 적 1마리당 1개 블록 모델, 최대 동시 적 24마리, 그림자·추가 조명·맵 장식·외곽선·파티클·미니맵 제거. AI 경로 판단은 12Hz, 적 겹침 보정은 8Hz로 묶어 CPU 사용량도 줄인다.
- 낮음: 45 FPS 목표와 동적 해상도를 사용하는 일반 사무용 PC용 설정.
- 보통/높음: 그림자와 추가 조명을 사용하는 균형·고화질 설정.

낮음과 초저사양은 실제 FPS를 감시하고 내부 렌더 해상도를 자동 조정한다. 게임 규칙과 전체 웨이브 수는 유지하며 한 화면에 동시에 활성화되는 적 수만 품질별로 제한한다. 초저사양을 게임 도중 선택하면 기존의 상세 적 모델·맵 장식·아이템·무기 모델도 즉시 경량 모델로 교체된다.

## 적 AI 개선

- 정밀 내비게이션 격자와 실제 적 크기에 맞춘 공유 비용 지도를 사용해 벽 뒤의 플레이어까지 통로를 따라 우회한다.
- 셀 중심뿐 아니라 이웃 셀 사이의 이동 구간 전체를 충돌 검사한다. 얇은 벽과 모서리를 가로질러 연결됐다고 오판하던 링크를 제거했다.
- 적 크기가 다른 적들이 경로 캐시를 서로 덮어쓰던 문제를 제거했다.
- 흐름 지도에 확정된 다음 칸을 기록하고 실제 시야가 확보된 경유지만 바라보므로 벽면에 붙어 비비지 않는다.
- 비용이 비슷한 양쪽 우회로에는 적 ID를 기준으로 분산해 한쪽 벽면에 전부 뭉치지 않는다.
- 플레이어 주변에 적 종류·ID별 공격 슬롯을 예약한다. 러너는 이동 방향의 측면을 노리고, 실드 좀비는 전열을 만들며, 균열술사는 원거리 간격을 유지해 같은 벽 정면에 전부 쌓이지 않는다.
- 플레이어 이동 방향을 짧게 예측해 코너 추적을 강화한다.
- 영구 벽은 통로를 따라 우회한다.
- 설치한 파괴 가능 벽은 우회 비용과 파괴 비용을 비교한다.
- 통로가 막혔거나 파괴가 더 빠르면 벽을 공격 목표로 고정해 근접 공격, 원거리 공격 또는 자폭으로 부순다.
- 공용 비용 지도가 실패하는 특수 상황에는 최대 18,000셀 장거리 A*와 벽 접선 추적을 사용한다.
- 막힘을 감지해 만든 A* 경로는 벽 끝을 통과할 때까지 유지한다. 저비용 흐름 지도가 다음 틱에 정밀 경로를 지우던 문제를 제거했다.
- 이동 속도뿐 아니라 플레이어까지의 실제 거리 감소량도 감시해, 벽면을 왕복하는 적을 막힘 상태로 판정한다.
- 플레이어 구역으로 실제 연결되지 않은 외벽 포켓은 적 스폰 후보에서 제외한다.
- 260셀보다 긴 경로도 시작점까지 완전히 복원하며, 저장 경로는 실제 직선 충돌 검사를 통과하는 구간만 압축한다.
- Thin Line 벽 우회, 얇은 벽 링크, 260셀 초과 경로, Abyss Citadel 16개 스폰 연결, A* 경로 유지, 격리 포켓 스폰 차단을 자동 회귀 테스트한다.

## 모바일 플레이

- 터치 기기를 자동 감지하고 모바일 FPS형 이동 스틱, 자유 시점 영역, 좌·우 발사, ADS 발사, 조준, 점프, 달리기 잠금, 재장전, 회복, 무기 변경, 일시정지 버튼을 표시한다.
- 스틱을 위로 끝까지 밀면 별도의 달리기 버튼을 동시에 누르지 않아도 자동 달리기한다.
- 좌·우 발사 버튼을 누른 채 드래그해 사격과 시점 조작을 동시에 할 수 있고, ADS 발사는 한 버튼으로 조준과 발사를 함께 수행한다.
- 조준은 한 번 눌러 유지하거나 누르는 동안만 유지하도록 선택할 수 있다.
- 게임 시작과 계속하기에서 전체 화면과 가로 화면 고정을 요청한다. 화면 잠금 API를 지원하지 않는 브라우저에서는 세로 화면을 차단하고 가로로 돌리라는 안내를 표시한다.
- 시작 화면과 일시정지 화면의 `모바일 조작 설정`에서 각 액션 버튼의 기능·시점 감도·크기·투명도를 변경할 수 있다.
- `터치키 위치 편집`에서 스틱과 모든 버튼을 직접 끌어 배치하며, 설정은 해당 기기의 브라우저에 자동 저장된다.
- 웹 앱 매니페스트에도 전체 화면·가로 방향을 지정해 홈 화면에 추가해 실행할 때 같은 모바일 구성을 사용한다.
- 모바일에서도 `입력 장치`를 `키보드·마우스`로 바꾸면 터치키를 끄고 블루투스/USB 키보드와 마우스 포인터 잠금을 사용한다. `자동 감지` 또는 `터치 조작`으로 언제든 돌아갈 수 있다.

- 설정 화면의 모든 동작 버튼을 같은 높이·모서리·간격의 공통 디자인으로 통일하고 의미에 따라 강조색만 구분했다.
- `게임 종료 후 메인으로`를 확인하면 실행 중인 월드와 타이머를 완전히 종료하고 문서를 다시 시작해 최초 로딩 화면과 메인 메뉴로 돌아간다.

## v46 모바일 반응형 UI·업그레이드 확인

- 모바일 가로 화면의 실제 너비와 높이에 맞춰 메인 메뉴, 일시정지 설정, 모바일 조작 설정, 키보드·마우스 설정의 여백·글자·입력칸·버튼 배치를 자동 압축한다.

## 스태미너 및 보상 수정

- 달리는 동안 스태미너가 소모되고 0이 되면 Shift를 누르고 있어도 걷는다.
- 0에서 Shift를 계속 누르면 회복하지 않으며, Shift를 놓으면 걷거나 가만히 서 있을 때 천천히 회복한다.
- 정지 중 회복은 걷는 중보다 12% 빠르다.
- 기본 회복량을 초당 10으로 조정했다. 기본 최대치 100을 0부터 채우는 데 10초가 걸린다.
- `스태미너 회복 +20%` 보상을 얻으면 현재 회복 속도에 20%씩 누적 적용된다.
- 스태미너가 조금이라도 회복되면 그 수치만큼 즉시 다시 달릴 수 있다.
- 조준 중에도 걷기·정지 상태 모두 회복되고, 조준 이동 속도는 일반 걷기보다 느리다.
- 벽에 막혔거나 공중에 있어 실제 지상 이동이 없으면 스태미너를 소모하지 않는다.
- 포커스 또는 포인터 잠금을 잃으면 Shift를 포함한 입력을 초기화해 키가 눌린 채 남지 않는다.
- `설치 벽 체력 +25%` 보상은 Wave 6에 WALL이 실제 해금된 이후에만 후보로 등장한다.

## GitHub Pages 공개 배포

`.github/workflows/pages.yml`이 `main` 브랜치의 `client` 폴더를 GitHub Pages에 자동 배포한다. 공개 저장소에 프로젝트를 올린 뒤 저장소의 `Settings → Pages → Source`를 `GitHub Actions`로 지정하면, Actions의 배포가 끝난 후 표시되는 주소에서 별도 설치 없이 게임을 실행할 수 있다. 브랜치 루트 방식으로 게시해도 루트 `index.html`이 즉시 `client/` 게임으로 이동시킨다.

공개 웹판은 브라우저용 `client`만 배포하며 Windows용 `BOXHEAD.exe`와 오프라인 실행 파일은 저장소의 릴리스 또는 압축본으로 제공할 수 있다.

## 개발용 빌드

완성 실행본은 이미 빌드되어 있다. 소스를 수정할 때만 Node.js 18 이상에서 다음 명령을 사용한다.

```bash
npm install
npm run build
```

선택 사항인 2인 협동 서버를 시험할 때만 `npm start`를 실행하고 `http://localhost:3000`으로 접속한다.
